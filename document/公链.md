# 公链入门指南

## 2.1 Layer 1

**Layer 1 网络**是底层区块链的别称。币安智能链 (BNB)、以太坊 (ETH)、比特币 (BTC) 和 Solana 都属于 Layer-1 协议。我们称之为 Layer 1，是因为它们是所在生态系统中的主要网络。与之相对的链下解决方案和 Layer 2 解决方案都是在主链上搭建的。

换言之，Layer 1 协议能够在自身的区块链上处理并完成交易，同时自带用于支付交易费用的原生代币。

### 常见的 Layer 1 区块链

| 区块链 | 原生代币 | 主要特点 |
|--------|----------|----------|
| **比特币 (Bitcoin)** | BTC | 首个加密货币，价值存储 |
| **以太坊 (Ethereum)** | ETH | 智能合约平台，DeFi 和 NFT 生态 |
| **币安智能链 (BNB Chain)** | BNB | 低费用，高吞吐量 |
| **Solana** | SOL | 高性能，低延迟 |
| **Cardano** | ADA | 学术研究驱动，权益证明 |
| **Avalanche** | AVAX | 高 TPS，子网架构 |
| **Polkadot** | DOT | 跨链互操作性 |
| **Cosmos** | ATOM | 区块链互联网 |

### Layer 1 的核心特点

| 特点 | 说明 |
|------|------|
| **独立区块链** | 拥有自己的主网和共识机制 |
| **原生代币** | 用于支付交易费用 (Gas) |
| **自主验证** | 在网络自身完成交易验证和结算 |
| **安全性** | 依赖自身的共识机制保障安全 |

### Layer 1 的挑战

Layer 1 区块链面临"不可能三角"的挑战：

```
        去中心化
           /\
          /  \
         /    \
        /      \
       /________\
   安全性        可扩展性
```

- **去中心化**：网络由大量独立节点组成
- **安全性**：抵御攻击的能力
- **可扩展性**：处理大量交易的能力

大多数 Layer 1 为了保障去中心化和安全性，不得不牺牲可扩展性，导致交易速度慢、费用高。

---

## 2.2 Layer 2

**Layer 2** 指基于底层区块链（Layer 1 网络）的链下网络、系统或技术，目的是为了扩展底层区块链网络。Layer 2 网络可以提升任何底层区块链的吞吐量以及其他性能。

### Layer 2 的核心价值

Layer 2 网络、系统或技术的核心价值是能够**利用底层区块链的安全性**。其交易数据必须以某种形式被底层区块链网络验证并确认。

> ⚠️ **注意**：按照这个标准来看，**侧链不属于 Layer 2**，因为侧链通常会部署自己的共识机制和验证节点，因此拥有独立于底层区块链的安全机制。

### 为什么需要 Layer 2？

一些区块链为了保障去中心化水平和安全性而牺牲了可扩展性，这类区块链可以利用 Layer 2 来：

- ✅ 提高交易吞吐量
- ✅ 降低交易成本
- ✅ 快速执行交易
- ✅ 实现可扩展性
- ✅ **无需牺牲去中心化水平或安全性**

---

## L2 解决方案的运行机制

> ℹ️ **注**：大多数 Layer 2 解决方案仍处于早期阶段，其中许多要素还有待验证。

### 基本架构

Layer 2 解决方案大致可以分为两个部分：

| 组件 | 功能 |
|------|------|
| **Layer 2 网络** | 负责处理交易的网络 |
| **Layer 1 智能合约** | 部署在底层区块链上，负责解决分歧，验证共识 |

```
┌─────────────────────────────────────┐
│           Layer 1 区块链             │
│  ┌─────────────────────────────┐    │
│  │      验证智能合约            │    │
│  │  - 保存并释放资金            │    │
│  │  - 验证证明                 │    │
│  │  - 解决分歧                 │    │
│  │  - 最终确认交易             │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
                  ↑↓
┌─────────────────────────────────────┐
│           Layer 2 网络               │
│  - 快速执行交易                      │
│  - 链下计算                          │
│  - 提交可验证证明                   │
└─────────────────────────────────────┘
```

### 工作机制

1. **快速执行**：Layer 2 网络上可以快速执行交易并开展计算
2. **提交证明**：向底层区块链提交可验证的加密证明
3. **验证确认**：Layer 1 智能合约验证证明，确认状态变更

不同 Layer 2 网络的区别：
- 有些**提前**创建证明并提交至底层区块链
- 有些**之后**创建追溯性的证明

### 智能合约的核心功能

不同 Layer 2 网络在底层区块链上的智能合约实现方式有所不同，但核心功能是一样的：

| 功能 | 说明 |
|------|------|
| **保存并释放资金** | 转账至 Layer 2 |
| **验证证明** | 收到 Layer 2 提交的证明进行验证 |
| **解决分歧** | 处理争议情况 |
| **最终确认** | 确认交易完成 |

## Layer 2 实现方案

为了更好地理解这两种机制，我们可以探讨目前的两种 Layer 2 实现方案：

### 1. 支付通道 (Payment Channels)

支付通道允许用户在链下进行多次交易，只有最终状态提交到链上。

**特点：**
- 适合高频小额支付
- 交易即时确认
- 仅开启和关闭通道需要链上交易

**代表项目：**
- 比特币闪电网络 (Lightning Network)
- 以太坊雷电网络 (Raiden Network)

### 2. Rollups

**Optimistic Rollups** 和 **ZK-Rollups** 可以在链下执行智能合约状态变更，并在链上进行验证，以此提升区块链的吞吐量并降低成本。

#### Rollups 的三种扩容方式

| 方式 | 说明 |
|------|------|
| **链下执行** | Rollups 在链下执行交易，底层区块链只需计算轻量化的交易证明 |
| **打包交易** | 将交易数据打包提交至区块链，链上 gas 费分摊到各个交易中 |
| **减少验证节点** | 最少只需一个诚实的验证节点即可证明交易有效性 |

---

## Rollups 详解

### 🔹 链下执行

Rollups 的一个关键特质就是在**链下执行交易**。这意味着 Layer 2 网络可以代替底层区块链处理与其他用户或智能合约的交易。

**优势：**
- ✅ 验证节点数量较少
- ✅ 硬件更强大
- ✅ 大幅提升交易吞吐量

**Layer 1 的工作：**
```
1. 计算提交至智能合约的证明
2. 验证 Layer 2 网络中的活动
3. 将未执行的原始交易数据作为 calldata 储存
```

> 💡 **简而言之**：区块链不再需要执行那么多计算任务，也不需要储存那么多 Layer 2 上的交易数据，因此可以降低每笔交易的成本。

**Optimistic Rollups 特别说明：**
> 只有当分歧出现时才需要验证

---

### 🔹 打包交易

另外一种降低成本的方式就是**将交易打包**。

**比喻理解：**

| 传统方式 | Rollups 方式 |
|----------|-------------|
| 每笔交易是单独的礼物，放在不同的盒子里 | 许多礼物放到一个大盒子里 |
| 每寄出一个礼物需要支付一笔运费 | 只需付一次运费 |
| 运费高 | 每个礼物分摊运费，成本低 |

**技术解释：**

Rollups 将原始交易数据打包成 **calldata**。

| 对比项 | 链上交易 | Rollup 交易 |
|--------|----------|-------------|
| 验证方式 | 逐笔验证 | 打包验证 |
| 数据存储 | 单独存储 | 打包存储 |
| Gas 费用 | 每笔独立支付 | 多笔分摊 |

> 💡 **核心逻辑**：主链上的一个打包交易中储存了多笔 rollup 交易。

---

### 🔹 减少验证节点数量

Layer 2 可以利用底层区块链的安全保障和去中心化水平。

**核心概念：**
> Layer 2 必须向底层区块链提交某种证明，以证明其发起的状态变更是有效的。

**Rollups 的验证节点特点：**

| 特点 | 说明 |
|------|------|
| **数量较少** | 减少 Layer 2 网络中负责执行交易的验证节点数量 |
| **获得许可** | 验证节点可以是获得许可的实体 |
| **硬件强大** | 拥有更强大的硬件，更快速度和更低成本 |
| **不负责共识** | 共识由底层区块链达成 |
| **安全性保障** | 底层区块链智能合约验证 Layer 2 提交的证明 |

---

## Layer 2 方案对比

| 特性 | Optimistic Rollups | ZK-Rollups | 支付通道 |
|------|-------------------|------------|----------|
| **验证方式** | 乐观假设，挑战期验证 | 零知识证明即时验证 | 链下签名，链下结算 |
| **提款时间** | 7 天挑战期 | 即时 | 即时 |
| **智能合约支持** | 完整支持 | 有限支持 | 仅支付 |
| **Gas 成本** | 较低 | 最低 | 最低 |
| **代表项目** | Arbitrum, Optimism | zkSync, StarkNet | 闪电网络 |

## 2.3 侧链

### 侧链定义

**侧链**是独立于主区块链（也称为主链）运行的并行区块链。二者通常通过**双向跨链桥**连接，从而实现主链和侧链之间无障碍划转数字资产或代币。

```
┌─────────────────┐         ┌─────────────────┐
│     主链         │◄───────►│     侧链         │
│   (Main Chain)  │  跨链桥  │  (Sidechain)    │
│                 │         │                 │
│   - 以太坊       │         │   - Polygon     │
│   - 比特币       │         │   - BSC         │
└─────────────────┘         └─────────────────┘
```

### 侧链 vs Layer 2

| 特性 | 侧链 | Layer 2 |
|------|------|----------|
| **安全性** | 独立的安全机制 | 依赖主链安全 |
| **共识机制** | 自有共识机制 | 使用主链共识 |
| **验证节点** | 独立验证节点 | 主链验证 |
| **归属关系** | 不属于 Layer 2 | 属于 Layer 2 |

> ⚠️ **重要区别**：侧链**不属于 Layer 2**，因为侧链拥有独立于底层区块链的安全机制。

---

### 侧链的优缺点

侧链与任何创新技术一样，也有其优势和局限性。

#### ✅ 优点

| 优点 | 说明 |
|------|------|
| **可扩展性** | 将一些交易处理转移到主链之外，从而减少拥塞并增强性能 |
| **灵活性** | 具有自主性，支持尝试不同的共识机制和参数，促进创新并实现可定制性 |
| **EVM 兼容性** | EVM 兼容的侧链为开发人员提供无障碍的迁移支持，可在侧链上部署现有的以太坊智能合约 |

**详细说明：**

1. **可扩展性**
   - 交易处理在主链外进行
   - 减少主链拥塞
   - 提升整体网络性能

2. **灵活性**
   - 可自定义共识机制
   - 可调整区块参数
   - 支持创新和实验

3. **EVM 兼容性**
   - 无缝迁移以太坊 DApp
   - 复用现有开发工具
   - 降低开发成本

---

#### ❌ 缺点

| 缺点 | 说明 |
|------|------|
| **去中心化妥协** | 实现高吞吐量通常需要牺牲一定程度的去中心化，可能导致权力集中在少数验证节点上，影响链的安全性 |
| **安全问题** | 侧链对自身安全负责，独立性可能带来更多的安全风险 |
| **复杂性** | 实施和维护需要大量精力和资源，初始设置复杂，持续维护需求可能影响应用 |

**详细说明：**

1. **去中心化妥协**
   - ⚠️ 验证节点数量较少
   - ⚠️ 权力可能集中
   - ⚠️ 安全性受影响

2. **安全问题**
   - 🔒 侧链自行负责安全
   - 🔒 不会直接影响主链
   - 🔒 但独立运行风险更高

3. **复杂性**
   - 🛠️ 初始设置复杂
   - 🛠️ 维护成本高
   - 🛠️ 需要专业团队

---

### 常见的侧链项目

| 侧链 | 主链 | 共识机制 | 特点 |
|------|------|----------|------|
| **Polygon (MATIC)** | 以太坊 | PoS | EVM 兼容，低费用 |
| **BNB Chain** | 独立 | PoSA | 高吞吐量，低成本 |
| **Ronin** | 以太坊 | PoA | 游戏专用侧链 |
| **SKALE** | 以太坊 | PoS | 弹性侧链网络 |
| **Liquid Network** | 比特币 | Federated | 比特币侧链 |

---

## 总结对比

| 层级 | 功能 | 安全性 | 代表项目 |
|------|------|--------|----------|
| **Layer 1** | 底层区块链，提供安全性和去中心化 | 自身共识 | BTC, ETH, BNB, SOL |
| **Layer 2** | 扩展方案，利用主链安全 | 依赖主链 | Arbitrum, Optimism, zkSync |
| **侧链** | 并行链，独立运行 | 独立安全 | Polygon, Ronin, SKALE |

### 选择建议

| 需求 | 推荐方案 |
|------|----------|
| 最高安全性 | Layer 1 主链 |
| 低成本 + 安全性 | Layer 2 |
| 高性能 + 灵活性 | 侧链 |
| 开发便捷性 | EVM 兼容侧链 |

---

## 整体总结

**Web3 基础设施三层架构：**

```
        ┌─────────────────┐
        │   Layer 1       │  ← 安全基石
        │  (底层区块链)    │
        └────────┬────────┘
                 │
        ┌────────┼────────┐
        │        │        │
        ▼        ▼        ▼
   ┌────────┐ ┌────────┐ ┌────────┐
   │ Layer 2│ │ Layer 2│ │ 侧链   │
   │(Rollup)│ │(通道)  │ │        │
   └────────┘ └────────┘ └────────┘
```

**核心价值：**
- 🔐 **Layer 1**：提供安全性和去中心化
- ⚡ **Layer 2**：利用主链安全，提升性能
- 🔗 **侧链**：独立运行，灵活可扩展
